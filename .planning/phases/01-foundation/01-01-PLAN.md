Read `frontend/src/app/globals.css`. It MUST contain `@import "tailwindcss"` as the first line (Tailwind v4 syntax). If it contains `@tailwind base;`, `@tailwind components;`, or `@tailwind utilities;`, the project scaffolded with Tailwind v3 — replace those three lines with `@import "tailwindcss"`.

After the import line, add the custom urgency pulse animation that Phase 4 will need. Replace the entire contents of `globals.css` with:

```css
@import "tailwindcss";

@theme {
  --animate-urgency-pulse: urgency-pulse 1s ease-in-out infinite;
}

@keyframes urgency-pulse {
  0%, 100% { opacity: 1; }
  50%       { opacity: 0.6; }
}
```

NOTE: Do NOT add `.animate-urgency-pulse` as a separate CSS class — in Tailwind v4, defining `--animate-urgency-pulse` inside `@theme` automatically generates the `animate-urgency-pulse` utility class. If it turns out Tailwind v4 does NOT auto-generate it from `@theme`, add this below the `@keyframes` block as a fallback:

```css
.animate-urgency-pulse {
  animation: urgency-pulse 1s ease-in-out infinite;
}
```

**Step 5: Write the root layout with ThemeProvider.**

Overwrite `frontend/src/app/layout.tsx` with:

```typescript
import type { Metadata } from 'next'
import { ThemeProvider } from 'next-themes'
import './globals.css'

export const metadata: Metadata = {
  title: 'ComputeBank',
  description: 'Compute asset management dashboard',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body>
        <ThemeProvider attribute="class" defaultTheme="dark" enableSystem={false}>
          {children}
        </ThemeProvider>
      </body>
    </html>
  )
}
```

Key points:
- `suppressHydrationWarning` on `<html>` prevents the next-themes hydration warning
- `defaultTheme="dark"` enforces dark mode (no toggle per CONTEXT.md decision)
- `enableSystem={false}` ignores OS preference — always dark

**Step 1: Create the DashboardNav sidebar component.**

Create `frontend/src/components/layout/DashboardNav.tsx` (create `layout/` subdirectory inside `components/`):

```typescript
import Link from 'next/link'
import { LayoutDashboard, Layers, History } from 'lucide-react'

const NAV_ITEMS = [
  { href: '/',        label: 'Dashboard', icon: LayoutDashboard },
  { href: '/leases',  label: 'Leases',    icon: Layers },
  { href: '/history', label: 'History',   icon: History },
]

export function DashboardNav() {
  return (
    <nav className="w-56 min-h-screen border-r border-border flex flex-col gap-1 p-4 bg-background">
      <div className="px-3 py-2 mb-2">
        <span className="text-lg font-semibold tracking-tight text-foreground">
          ComputeBank
        </span>
      </div>
      {NAV_ITEMS.map(({ href, label, icon: Icon }) => (
        <Link
          key={href}
          href={href}
          className="flex items-center gap-3 px-3 py-2 rounded-md text-sm text-muted-foreground hover:text-foreground hover:bg-accent transition-colors"
        >
          <Icon className="w-4 h-4" />
          {label}
        </Link>
      ))}
    </nav>
  )
}
```

IMPORTANT: No `"use client"` directive — this is a Server Component. `lucide-react` is safe to use in Server Components.
